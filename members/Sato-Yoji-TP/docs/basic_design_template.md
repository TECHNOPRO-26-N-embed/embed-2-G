## 1. システム全体像

### 1-1. ブロック構成（入力 → 処理 → 出力）

> ※ Arduinoを中心に「何が入力で何が出力か」をテキストの図で整理します。

```
【入力】                               【Arduino UNO R3】                          【出力】
Button（押下/長押し） ─────────────→┐                                       ┌──→ 4 Digit-7 Segment（SS.hh）
Ultrasonic Sensor（3-5cm検知） ──→┤  状態管理・時刻更新・入力判定・表示多重化  ├──→ 1 Digit-7 Segment（分表示: 任意）
                                    └───────────────────────────────────────┘   └──→ Red LED（start/stop時1秒点灯）
```

補足説明（図で表現しきれない場合）：
- ButtonとUltrasonic Sensorは同じstart/stopトリガとして扱う。
- 長押し（1秒以上）はreset専用入力として扱う。
- 表示更新と入力監視は同時進行のため、全処理をmillis()ベースで実装する。

---

### 1-2. 状態遷移（ガジェットの「ふるまい」設計）

> ※ ガジェットが「どんな状態を持つか」を定義します。
> この設計が loop() の骨格になります。

**記入例：**
```
[電源ON / 初期化]
  ↓（初期化完了）
[待機中] ──（センサー検知）──→ [計測中] ──（閾値超え）──→ [警告中]
   ↑                                                            │
   └────────────────────（リセットボタン）────────────────────────┘
```

**↓ 自分の状態遷移を書いてください**
```
[電源ON / 初期化]
  ↓（初期化完了）
[停止表示(00.00)] ──（Button短押し or Sensor検知）──→ [計測中]
   ↑                          │
   │                          ├─（Button短押し or Sensor検知）──→ [停止表示(保持)]
   │                          └─（59.99到達）──────────────────→ [上限停止(59.99固定)]
   │
   └────────────（停止表示(保持)でButton短押し or Sensor検知）────────────→ [計測中]

※ 全状態（停止表示(00.00), 計測中, 停止表示(保持), 上限停止）から
   （Button長押し>=1s）で [停止表示(00.00)] に遷移


```

**各状態の意味：**

| 状態名 | この状態のとき、何をしているか | LEDやブザーなど出力の様子 |
|:--|:--|:--|
| （例）待機中 | センサー値を100msごとに監視する | 緑LED：500msで点滅 |
| 停止表示(00.00) | リセット後の初期値を表示し、入力待ちする | LED消灯 |
| 計測中 | 10msごとに計測値（centisecond）を更新して表示する | start時1秒点灯、その後消灯 |
| 停止表示(保持) | 計測値を保持して表示し、再スタート入力を待つ | stop時1秒点灯、その後消灯 |
| 上限停止(59.99固定) | 安全要件に従い59.99で固定し、reset待ちする | LED消灯 |

### 2-3. タイミング設計（delay vs millis）

> ※ `delay(N)` はN ms間、**すべての処理**が止まります。
> 複数の処理を並行させたい場合は `millis()` を使います。

| 処理 | 必要な周期 | delay / millis | 判断の理由 |
|:--|:--|:--|:--|
| （例）LED点滅 | 500ms | **millis** | ボタン監視と並行するため |
| （例）センサー読み取り | 100ms | **millis** | 常時監視が必要なため |
| ボタン監視（デバウンス） | 1-5ms相当 | millis | 長押し判定と同時処理するため |
| 超音波センサー読み取り | 50ms | millis | 連続測定しつつ表示を止めないため |
| 計時カウントアップ | 10ms | millis | 0.01秒表示に合わせるため |
| 7セグ表示の多重化 | 2ms/桁 | millis | ちらつきを抑えながら入力監視を継続するため |
| LED 1秒点灯制御 | 1ms監視 | millis | delayを使わず他処理を止めないため |

> [!CAUTION]
> `delay()` を使う場合、その間はボタン入力もセンサー読み取りも**すべて止まります**。

## 4. 異常系・エラー処理の方針

| 異常ケース | 発生条件 | 検知方法 | 対応方針 |
|:--|:--|:--|:--|
| チャタリング | ボタンを押した直後 | 前回変化から30ms未満の変化 | 変化を無視する |
| センサー誤検知 | 背景物体で距離が一瞬閾値内 | 閾値内判定が1回のみ | 2回連続一致で有効化 |
| センサー異常値 | 距離値が0cmまたは400cm超 | 範囲外チェック | 今回値を破棄し前回値を保持 |
| 多重トリガ | start/stop直後に再入力 | 最終トリガから300ms未満 | 不感時間として入力無効 |
| 上限超過 | 計測値が59.99を超える | elapsedCs >= 5999 | 59.99固定状態へ遷移しreset待ち |
