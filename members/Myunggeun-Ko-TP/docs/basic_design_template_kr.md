## 1. 시스템 전체상

### 1-1. 블록 구성(입력 → 처리 → 출력)

> ※ Arduino를 중심으로 “무엇이 입력이고 무엇이 출력인지”를 텍스트 그림으로 정리합니다.

**기입 예:**
```text
【입력】                 【Arduino UNO R3】            【출력】
택트 스위치 ─────→┐                         ┌──→ LED(빨강·초록)
초음파 센서 ─────→┤   상태 관리·판정 처리    ├──→ 패시브 부저
                  └─────────────────────────┘
```

**↓ 자신의 시스템을 작성하세요**
```text
【입력】                 【Arduino UNO R3】            【출력】
PIR 인체감지 센서 ───→┐                         ┌──→ DC 모터(선풍기)
                      │   사람 감지·타이머 관리 ├──→ 4Digit 7-Segment Display
                      │   PWM값 제어 처리        └──→ LED(동작 상태 표시)
                      └─────────────────────────┘
```

보충 설명(그림으로 표현하기 어려운 경우):
PIR 인체감지 센서로 사람의 움직임을 감지하고, Arduino UNO R3가 감지 상태를 판정한다. 사람을 감지한 경우 DC 모터를 PWM 제어로 회전시키고, PWM값을 4Digit 7-Segment Display에 표시한다. 20초 동안 감지가 없는 경우 모터를 정지하고, 7세그먼트 디스플레이에 “OFF”를 표시한다.

### 1-2. 상태 전이(가젯의 “동작 방식” 설계)

> ※ 가젯이 “어떤 상태를 가지는지”를 정의합니다.
> 이 설계가 loop()의 골격이 됩니다.

**기입 예:**
```text
[전원 ON / 초기화]
  ↓（초기화 완료）
[대기중] ──（센서 감지）──→ [계측중] ──（임계값 초과）──→ [경고중]
   ↑                                                            │
   └────────────────────（리셋 버튼）────────────────────────┘
```

**↓ 자신의 상태 전이를 작성하세요**
```text
[전원 ON / 초기화]
  ↓（초기화 완료）
[대기중] ──（사람 감지）──→ [동작중] ──（20초 동안 미감지）──→ [정지중]
   ↑                                                         │
   └──────────────（다시 사람 감지）─────────────────────────────┘
```

**각 상태의 의미:**

| 상태명 | 이 상태일 때 무엇을 하고 있는가 | LED나 부저 등 출력 상태 |
|:--|:--|:--|
| 대기중 | PIR 센서 값을 감시하며 사람을 감지한다 | LED: 소등, 7세그: OFF 표시 |
| 동작중 | 모터를 PWM 제어로 회전시킨다. PWM값을 점점 증가시킨다 | LED: 점등, 7세그: PWM값 표시 |
| 정지중 | 모터를 정지하고, 다시 사람을 감지할 때까지 대기한다 | LED: 소등, 7세그: OFF 표시 |

> [!TIP]
> 모든 상태에서 “빠져나갈 수 있는 경로”가 있는지 확인하십시오.
> “이 버튼을 누르면 반드시 리셋으로 돌아간다”가 보장되어 있습니까?

### 2-3. 타이밍 설계(delay vs millis)

> ※ `delay(N)`은 N ms 동안 **모든 처리**가 멈춥니다.
> 여러 처리를 병행하고 싶은 경우에는 `millis()`를 사용합니다.

| 처리 | 필요한 주기 | delay / millis | 판단 이유 |
|:--|:--|:--|:--|
| (예) LED 점멸 | 500ms | **millis** | 버튼 감시와 병행하기 위해 |
| (예) 센서 읽기 | 100ms | **millis** | 상시 감시가 필요하기 때문에 |
|  |  |  |  |
|  |  |  |  |

| 처리 | 필요한 주기 | delay / millis | 판단 이유 |
|:--|:--|:--|:--|
| PIR 센서 읽기 | 100ms | **millis** | 사람 감지를 상시 감시하기 위해 |
| PWM값 증가 처리 | 100ms | **millis** | 선풍기를 서서히 강하게 하면서 센서 감시도 계속하기 위해 |
| 미감지 시간 확인 | 항상 / 100ms마다 | **millis** | 마지막 감지 후 20초가 경과했는지 확인하기 위해 |
| 7세그먼트 디스플레이 표시 업데이트 | 100ms | **millis** | PWM값이나 OFF 표시를 정기적으로 업데이트하기 위해 |
| LED 상태 표시 | 100ms | **millis** | 동작중·정지중 상태를 바로 확인할 수 있도록 하기 위해 |

> [!CAUTION]
> `delay()`를 사용하는 경우, 그동안 버튼 입력도 센서 읽기도 **모두 멈춥니다**.

## 3. 하드웨어 설계

### 3-1. 부품 목록과 핀 연결

> ※ 디지털 핀: D2~D13 / 아날로그 핀: A0~A5
> 특수 용도: D0/D1(시리얼 통신·사용 금지), A4/A5(I2C), D10~D13(SPI)

| 부품명 | 모델명·종류 | Arduino 핀 | VCC / GND | 입력 / 출력 | 비고 |
|:--|:--|:--|:--|:--|:--|
| PIR 인체감지 센서 | HC-SR501 PIR Motion Sensor Module | D12 | 5V / GND | 입력 | 사람을 감지한다 |
| DC 모터 | Fan Blade and DC 3-6V Motor | L293D에 연결 | 외부 전원 / GND | 출력 | PWM 제어로 회전 속도를 변경 |
| 모터 드라이버 | L293D Motor Driver IC | D6, D7, D11 | 5V / GND | 출력 | 모터 제어용 |

## 7. AI 리뷰 기록

### Q1: 설계의 타당성 확인

> “이 기본설계서를 바탕으로 Arduino UNO R3에서 코드를 작성합니다. 핀 충돌, 라이브러리 충돌, 상태 전이 누락, delay()의 문제점은 있습니까?”

**AI의 답변(요약):**
- 현재 핀 배치에서는 큰 충돌은 확인되지 않는다
- D0/D1을 사용하지 않기 때문에 Serial 통신과의 충돌은 없다
- 7세그먼트 디스플레이와 L293D의 핀이 분리되어 있어 문제없다
- `delay()`를 많이 사용하면 PIR 센서 감시가 정지하므로 `millis()` 사용이 적절하다
- 상태 전이는 “대기중 → 동작중 → 정지중 → 대기중”의 흐름으로 정리되어 있다
- HC-SR501은 오검출이나 검지 지연이 있으므로 타이머 유지 처리가 유효하다

**대응한 내용:**

- 핀 배치와 상태 전이를 재확인했다
- PIR 센서 오검출 대책으로 타이머 제어를 추가했다

### Q2: 채터링·delay·노이즈 확인

> “이 설계에 채터링 대책은 포함되어 있습니까? delay()를 사용하는 부분에 문제가 있습니까? 아날로그 센서를 사용하는 경우 노이즈 대책이 필요합니까?”

**AI의 답변(요약):**
- 버튼을 사용하지 않으므로 채터링 대책은 불필요

- PIR 센서는 디지털 출력이므로 아날로그 노이즈 대책은 불필요
- PIR 센서는 짧은 시간의 오검출이 발생할 가능성이 있으므로, 일정 시간 감지를 유지하는 처리가 권장된다

**대응한 내용:**

- 버튼을 사용하지 않으므로 채터링 대책은 불필요하다고 확인했다
- PIR 센서의 오검출을 고려하여 일정 시간의 감지 유지 처리를 추가했다
