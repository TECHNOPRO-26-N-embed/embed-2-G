## 3. 기능 요구사항

### 3-1. 이 가젯이 할 수 있는 것 (필수 기능)
> ※ “~할 수 있다” 형태로 작성. 최소 3개 이상.

- 선풍기를 자동으로 사용할 수 있다
- 방에서 나가면 선풍기가 멈춘다
- 4자리 7세그먼트 디스플레이에 PWM 값(0000~0100)을 표시한다. 정지 시에는 “OFF”를 표시한다

### 3-2. 있으면 좋은 기능 (선택·여유가 있다면)

- 사람이 없는 시간이 길어지면 완전히 정지할 수 있다
- 작동 중에 LED로 상태를 확인할 수 있다

### 3-3. 하지 않는 것 (범위 외)
> ※ “~하지 않는다” 형태로 작성. AI가 멋대로 기능을 확장하는 것을 막기 위해 중요합니다.

- 온도에 따른 자동 조정은 하지 않는다
- 스마트폰과의 연동은 하지 않는다
- 음성 인식 기능은 추가하지 않는다

### 3-4. 비기능 요구사항 (선택)
> ※ 여유가 있다면 작성해주세요. AI는 이 부분을 무시하기 쉽습니다.

- **안전성:** 센서 값이 이상한 경우 선풍기를 정지시킨다
- **응답 시간:** 사람을 감지한 후 1초 이내에 반응한다

### Q2: 실현 가능성 확인
> “이 요구사항은 Arduino UNO R3와 ELEGOO 키트로 구현할 수 있나요? 어려운 요구사항이 있다면 알려주세요.”

## AI의 답변 (요약):
네, 현재 작성한 요구사항은 Arduino UNO R3 + ELEGOO 키트만으로 충분히 구현 가능합니다.

오히려 임베디드 개발 입문 실습으로는 난이도도 적절하고 구성도 좋습니다.

조금 어려울 수 있는 부분

### 1. PWM을 이용한 모터 속도 제어

가장 주의가 필요한 부분입니다.

Arduino에서는 PWM으로 속도 제어가 가능하지만,

소형 저가 DC 모터의 경우
- 속도 변화가 부드럽지 않을 수 있음
- 저속에서는 회전하지 않을 수 있음

이라는 가능성이 있습니다.

“풍량 1·2·3단계” 정도는 가능하지만,
“세밀한 속도 제어”는 조금 어려울 수 있습니다.

단계 제어를 추천합니다.

### 2. PIR 센서의 특성

HC-SR501은 생각보다 완벽하게 반응하지 않습니다.

특징:
- 움직임이 있을 때 감지하기 쉬움
- 가만히 있으면 감지하지 못할 수 있음
- 약간의 지연이 있음
- 오작동 감지가 있을 수 있음

예를 들어,
사람이 방에 있어도 움직이지 않으면 OFF 될 가능성이 있습니다.

해결 방법:
- 마지막 감지 후 30초 유지
- 일정 시간 ON 유지

같은 방식으로 대응합니다.

즉시 OFF는 추천하지 않습니다.

### 3. 4Digit 7-Segment Display 제어

구현은 가능하지만 초보자에게는 조금 어렵게 느껴질 수 있습니다.

이유:
- TM1637 타입인지
- 일반적인 12핀 타입인지

에 따라 코드가 크게 달라지기 때문입니다.

ELEGOO 키트의 7세그가
- TM1637 모듈인지
- 일반 FND인지

확인하는 것이 좋습니다.

TM1637 쪽이 훨씬 간단합니다.

## 대응한 내용 (범위 변경 등):

1. PWM을 이용한 모터 속도 제어  
    간단한 모터 제어를 사용하여, 작동 시 점점 강해지도록 합니다.  
    사람을 감지하면 PWM 값을 1부터 100까지 점점 증가시킵니다.

3. 4Digit 7-Segment Display 제어  
    12핀 방식으로 제어합니다. PDF 파일을 확인하고 이해할 필요가 있습니다.
