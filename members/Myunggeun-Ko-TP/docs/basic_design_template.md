## 1. システム全体像

### 1-1. ブロック構成（入力 → 処理 → 出力）

> ※ Arduinoを中心に「何が入力で何が出力か」をテキストの図で整理します。

**記入例：**
```
【入力】                 【Arduino UNO R3】            【出力】
タクトスイッチ ─────→┐                         ┌──→ LED（赤・緑）
超音波センサー ─────→┤   状態管理・判定処理    ├──→ パッシブブザー
                      └─────────────────────────┘
```

**↓ 自分のシステムを書いてください**
```
【入力】                 【Arduino UNO R3】            【出力】
PIR人感センサー ─────→┐                         ┌──→ DCモーター（扇風機）
                      │   人の検知・タイマー管理 ├──→ 4Digit 7-Segment Display
                      │   PWM値の制御処理        └──→ LED（動作状態表示）
                      └─────────────────────────┘



```

補足説明（図で表現しきれない場合）：
PIR人感センサーで人の動きを検知し、Arduino UNO R3が検知状態を判定する。人を検知した場合はDCモーターをPWM制御で回転させ、PWM値を4Digit 7-Segment Displayに表示する。20秒間検知がない場合はモーターを停止し、7セグメントディスプレイに「OFF」を表示する。

### 1-2. 状態遷移（ガジェットの「ふるまい」設計）

> ※ ガジェットが「どんな状態を持つか」を定義します。
> この設計が loop() の骨格になります。

**記入例：**
```
[電源ON / 初期化]
  ↓（初期化完了）
[待機中] ──（センサー検知）──→ [計測中] ──（閾値超え）──→ [警告中]
   ↑                                                            │
   └────────────────────（リセットボタン）────────────────────────┘
```

**↓ 自分の状態遷移を書いてください**
```
[電源ON / 初期化]
  ↓（初期化完了）
[待機中] ──（人を検知）──→ [動作中] ──（20秒間未検知）──→ [停止中]
   ↑                                                         │
   └──────────────（再度人を検知）─────────────────────────────┘


```

**各状態の意味：**


| 状態名 | この状態のとき、何をしているか                | LEDやブザーなど出力の様子    |
| :-- | :----------------------------- | :---------------- |
| 待機中 | PIRセンサーの値を監視して人を検知する           | LED：消灯、7セグ：OFF表示  |
| 動作中 | モーターをPWM制御で回転させる。PWM値を徐々に増加させる | LED：点灯、7セグ：PWM値表示 |
| 停止中 | モーターを停止し、再度人を検知するまで待機する        | LED：消灯、7セグ：OFF表示  |

> [!TIP]
> すべての状態から「抜け出せるルート」があるか確認してください。
> 「このボタンを押せば必ずリセットに戻る」が保証されていますか？

### 2-3. タイミング設計（delay vs millis）

> ※ `delay(N)` はN ms間、**すべての処理**が止まります。
> 複数の処理を並行させたい場合は `millis()` を使います。

| 処理 | 必要な周期 | delay / millis | 判断の理由 |
|:--|:--|:--|:--|
| （例）LED点滅 | 500ms | **millis** | ボタン監視と並行するため |
| （例）センサー読み取り | 100ms | **millis** | 常時監視が必要なため |
|  |  |  |  |
|  |  |  |  |
| 処理               | 必要な周期        | delay / millis | 判断の理由                      |
| :--------------- | :----------- | :------------- | :------------------------- |
| PIRセンサー読み取り      | 100ms        | **millis**     | 人の検知を常時監視するため              |
| PWM値の増加処理        | 100ms        | **millis**     | 扇風機を徐々に強くしながら、センサー監視も続けるため |
| 未検知時間の確認         | 常時 / 100msごと | **millis**     | 最後の検知から20秒経過したか確認するため      |
| 7セグメントディスプレイ表示更新 | 100ms        | **millis**     | PWM値やOFF表示を定期的に更新するため      |
| LED状態表示          | 100ms        | **millis**     | 動作中・停止中の状態をすぐ確認できるようにするため  |

> [!CAUTION]
> `delay()` を使う場合、その間はボタン入力もセンサー読み取りも**すべて止まります**。

## 4. 異常系・エラー処理の方針

| 異常ケース       | 発生条件                  | 検知方法                | 対応方針                           |
| :---------- | :-------------------- | :------------------ | :----------------------------- |
| PIRセンサーの誤検知 | ノイズや周囲環境で誤検知したとき      | 短時間でHIGH/LOWが繰り返される | 一定時間状態を維持し、すぐに停止しない            |
| 人を検知できない    | 人が動かずPIRセンサーが反応しない    | 20秒間検知信号が来ない        | タイマー満了後にモーターを停止する              |
| モーターが回転しない  | PWM値が低すぎる場合           | PWM出力後も回転が確認できない    | PWM値を徐々に上げて最低動作値以上で使用する        |
| 7セグメント表示異常  | 配線ミスやデータ送信失敗          | 表示値が崩れる・点灯しない       | OFF表示へ切り替えて配線確認を行う             |
| 電流過多        | モーターをArduinoへ直接接続した場合 | モーター動作時に不安定になる      | L293D Motor Driver ICを使用して保護する |
