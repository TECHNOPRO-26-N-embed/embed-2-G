## 1. 글로벌 변수 · 상수 설계

> ※ 기본설계서(2-1 데이터 설계)를 바탕으로, **자료형과 초기값까지** 결정합니다.

```text

  MOTOR_IN1_PIN    = 6     // L293D 모터 드라이버 IC
  MOTOR_IN2_PIN    = 7     // L293D 모터 드라이버 IC
  MOTOR_PWM_PIN    = 11    // L293D 모터 드라이버 IC(PWM 제어)

  SEG_DATA_PIN     = 8     // 4자리 7세그먼트 디스플레이
  SEG_CLOCK_PIN    = 9     // 4자리 7세그먼트 디스플레이
  SEG_LATCH_PIN    = 10    // 4자리 7세그먼트 디스플레이

  SEG_DIGIT1_PIN   = 2     // 세그먼트 자리 선택
  SEG_DIGIT2_PIN   = 3
  SEG_DIGIT3_PIN   = 4
  SEG_DIGIT4_PIN   = 5

  LED_PIN          = 13    // LED

// L293D Motor Driver IC 연결 변경

// DIRB   : D4 → D7
```

## 2. 각 함수의 상세설계

> ※ 기본설계서(2-2 함수 목록)에서 정의한 각 함수의 “내용”을 설계합니다.
> **의사코드**(한국어 + 처리 흐름)로 작성합니다. 실제 C++ 코드는 작성하지 않아도 됩니다.

---

### `setup()` — 초기화 처리

**↓ 자신의 setup() 을 설계하세요**

```text
【처리 흐름】

1. 핀 모드를 설정한다
   - PIR 인체감지 센서 D12 → INPUT
   - L293D Motor Driver IC D6, D7, D11 → OUTPUT
   - 4Digit 7-Segment Display D8, D9, D10 → OUTPUT
   - 7세그먼트 자리 선택 D2, D3, D4, D5 → OUTPUT
   - LED D13 → OUTPUT

2. 초기 상태를 설정한다
   - currentState 를 “대기중”으로 설정
   - pwmValue 를 0으로 설정
   - motorRunning 을 false 로 설정
   - lastDetectMillis 를 0으로 설정
   - lastPwmMillis 를 0으로 설정

3. 출력을 초기화한다
   - 모터를 정지한다
   - LED를 소등한다
   - 7세그먼트 디스플레이에 “OFF”를 표시한다

4. Serial.begin(9600) 을 실행한다
   - 디버그용으로 상태와 센서값을 확인할 수 있도록 한다
```

---

### `loop()` — 메인 루프

> ※ loop() 는 “상태별로 무엇을 할지”만 작성한다. 세부 처리는 각 함수에 맡긴다.

**↓ 자신의 loop() 를 설계하세요**

```text
【처리 흐름】

＜매 루프마다 실행할 것＞
  - 현재 시간을 가져온다: currentMillis = millis()
  - PIR 인체감지 센서 값을 읽는다
  - LED와 7세그먼트 디스플레이 표시를 업데이트한다

＜currentState 가 0(대기중)일 때＞
  - PIR 센서로 사람을 감지할 때까지 대기한다
  - 사람을 감지한 경우:
      - lastDetectMillis 를 현재 시각으로 업데이트한다
      - currentState 를 1(동작중)으로 변경한다

＜currentState 가 1(동작중)일 때＞
  - 모터를 PWM 제어로 회전시킨다
  - PWM 값을 점점 증가시킨다
  - 7세그먼트 디스플레이에 PWM 값을 표시한다
  - 사람을 감지한 경우:
      - lastDetectMillis 를 업데이트한다
  - 20초 동안 사람을 감지하지 못한 경우:
      - currentState 를 2(정지중)로 변경한다

＜currentState 가 2(정지중)일 때＞
  - 모터를 정지한다
  - 7세그먼트 디스플레이에 “OFF”를 표시한다
  - PIR 센서로 다시 사람을 감지한 경우:
      - lastDetectMillis 를 업데이트한다
      - currentState 를 1(동작중)으로 변경한다
```

### `readPirSensor()` — PIR 센서의 감지 상태를 읽는다

**basic_design.md 2-2 대응:** PIR 센서의 감지 상태를 읽는다

**인수:** 없음

**반환값:** bool : 사람을 감지했는지 여부

```text
【처리 흐름】
1. PIR 센서 입력값을 읽는다
2. HIGH 인 경우 true 를 반환한다
3. LOW 인 경우 false 를 반환한다

【에러 · 이상 케이스】
- 센서값이 불안정한 경우:
  바로 상태를 변경하지 않고 다음 루프에서 다시 확인한다
```

---

### `updateState()` — 상태를 전환한다

**basic_design.md 2-2 대응:** 센서값과 타이머를 기반으로 상태를 전환한다

**인수:** `detected`(bool) : 사람을 감지했는지 여부

**반환값:** void

```text
【처리 흐름】
1. detected 가 true 인 경우, lastDetectMillis 를 업데이트한다
2. 대기중에서 detected 가 true 인 경우, 동작중으로 변경한다
3. 동작중에서 20초 동안 미감지 상태인 경우, 정지중으로 변경한다

【에러 · 이상 케이스】
- 상태 번호가 예상 외의 값인 경우:
  currentState 를 0(대기중)으로 되돌린다
```

---

### `updateDisplay()` — 표시를 업데이트한다

**basic_design.md 2-2 대응:** 상태에 따라 PWM값 또는 OFF를 7세그에 표시한다

**인수:** `pwmValue`(int) : 표시할 PWM값

**반환값:** void

```text
【처리 흐름】
1. currentState 를 확인한다
2. 동작중인 경우 showPwmValue() 를 호출한다
3. 대기중 또는 정지중인 경우 showOff() 를 호출한다

【에러 · 이상 케이스】
- pwmValue 가 0 미만 또는 255 초과인 경우:
  0~255 범위로 보정해서 표시한다
```

---

### `startFan()` — 모터를 동작시킨다

**basic_design.md 2-2 대응:** 사람을 감지했을 때 모터를 동작시킨다

**인수:** `pwmValue`(int) : 모터에 출력할 PWM값

**반환값:** void

```text
【처리 흐름】
1. 모터 드라이버의 제어 핀을 회전 방향으로 설정한다
2. PWM 값을 모터에 출력한다
3. motorRunning 을 true 로 설정한다

【에러 · 이상 케이스】
- pwmValue 가 이상한 값인 경우:
  0~255 범위로 보정한 뒤 출력한다
```

---

### `stopFan()` — 모터를 정지시킨다

**basic_design.md 2-2 대응:** 20초 동안 감지가 없을 경우 모터를 정지시킨다

**인수:** 없음

**반환값:** void

```text
【처리 흐름】
1. 모터로 가는 PWM 출력을 0으로 설정한다
2. 모터 드라이버의 제어 핀을 정지 상태로 설정한다
3. motorRunning 을 false 로 설정한다

【에러 · 이상 케이스】
- 모터 정지 후에도 동작 상태가 남아있는 경우:
  pwmValue 를 0으로 되돌리고 상태를 정지중으로 설정한다
```

---

### `showPwmValue()` — PWM 값을 표시한다

**basic_design.md 2-2 대응:** 4자리 7세그먼트 디스플레이에 PWM값을 0000~0255로 표시한다

**인수:** `pwmValue`(int) : 표시할 PWM값

**반환값:** void

```text
【처리 흐름】
1. pwmValue 를 4자리 표시용 값으로 변환한다
2. 7세그먼트 디스플레이에 표시 데이터를 전송한다
3. 표시를 업데이트한다

【에러 · 이상 케이스】
- pwmValue 가 0 미만 또는 255 초과인 경우:
  표시 가능한 범위로 보정한다
```

---

### `showOff()` — OFF를 표시한다

**basic_design.md 2-2 대응:** 모터 정지 시 7세그먼트 디스플레이에 OFF를 표시한다

**인수:** 없음

**반환값:** void

```text
【처리 흐름】
1. 표시 내용을 “OFF”로 설정한다
2. 7세그먼트 디스플레이에 표시 데이터를 전송한다
3. 표시를 업데이트한다

【에러 · 이상 케이스】
- OFF 표시가 불가능한 경우:
  모든 자리를 소등한다
```

---

### `checkNoMotionTimeout()` — 미감지 시간을 확인한다

**basic_design.md 2-2 대응:** 마지막 감지 후 20초가 경과했는지 확인한다

**인수:** `lastDetectMillis`(unsigned long) : 마지막으로 사람을 감지한 시각

**반환값:** bool : 20초 경과 여부

```text
【처리 흐름】
1. 현재 시간을 millis() 로 가져온다
2. 현재 시간과 lastDetectMillis 의 차를 계산한다
3. 20초 이상 경과했으면 true 를 반환한다

【에러 · 이상 케이스】
- lastDetectMillis 가 미설정 상태인 경우:
  false 를 반환하여 바로 정지하지 않는다
```

---

**반환값:** void

```text
【처리 흐름】
1. state 를 확인한다
2. 동작중인 경우 LED를 점등한다
3. 대기중 또는 정지중인 경우 LED를 소등한다

【에러 · 이상 케이스】
- state 가 예상 외의 값인 경우:
  LED를 소등한다
```

### 3-2. millis() 를 사용한 타이머 관리

```text
【생각 방식】
  “이전에 실행한 시각”을 기록해두고,
  “현재 시각 − 이전 시각 ≥ 주기”이면 실행한다.

【처리 흐름(예: LED 점멸)】
  1. now = millis()
  2. now - lastMillis_LED >= LED_INTERVAL 인지 확인한다
  3. 조건을 만족한 경우:
     LED의 ON/OFF 를 전환하고 lastMillis_LED = now 로 설정한다
  4. 조건을 만족하지 않는 경우:
     아무것도 하지 않는다. 다음 루프에서 다시 확인한다.

【자신의 시스템에서 millis() 를 사용하는 처리】
  basic_design.md 2-3 의 타이밍 설계에서 전기하여 구체화한다.
```

---

### 3-3. 기타 중요 로직(선택)

```text
【로직명】
PIR 센서의 오검출 대책과 20초 정지 판정

【생각 방식】
PIR 센서는 사람의 움직임이나 주변 환경에 의해 일시적으로 HIGH/LOW 가 변할 수 있다.
따라서 한 번 LOW 가 되었다고 바로 선풍기를 정지하지 않고,
마지막으로 사람을 감지한 시각부터 20초가 경과했을 경우에 정지한다.

【처리 흐름】
1. PIR 센서 값을 읽는다
2. 사람을 감지한 경우, lastDetectMillis 를 현재 시각으로 업데이트한다
3. 사람을 감지하는 동안은 모터를 동작시키고 PWM 값을 표시한다
4. 사람을 감지하지 않는 상태가 계속된 경우,
   현재 시각과 lastDetectMillis 의 차를 확인한다
5. 20초 이상 경과한 경우,
   모터를 정지하고 7세그먼트 디스플레이에 “OFF”를 표시한다

【입력값과 출력값의 관계】

PIR 센서 = HIGH
→ 사람 감지
→ 모터 동작
→ LED 점등
→ 7세그먼트 디스플레이에 PWM 값 표시

PIR 센서 = LOW 그리고 20초 미만
→ 바로 정지하지 않음
→ 모터 동작 유지
→ PWM 값 표시 유지

PIR 센서 = LOW 그리고 20초 이상
→ 사람이 없다고 판단
→ 모터 정지
→ LED 소등
→ 7세그먼트 디스플레이에 “OFF” 표시
```

## 4. 디버그 출력 계획(선택)

> **【선택】** 함수 설계(Section 2)와 병행해서 작성하면 효과적입니다.
> “동작하지 않을 때 무엇을 확인하면 되는지”를 사전에 계획해둡니다.
> 구현 후에는 불필요한 Serial.println() 을 삭제합니다.

| No | 확인하고 싶은 내용 | 삽입할 함수 | Serial.println 내용 예시 |
|:---|:---|:---|:---|
| 1 | PIR 센서값이 올바르게 취득되는지 | `readPirSensor()` | `Serial.println(pirValue);` |
| 2 | 상태 전이가 올바르게 수행되는지 | `loop()` | `Serial.println(currentState);` |
| 3 | 사람을 감지한 시각이 업데이트되는지 | `updateState()` | `Serial.println(lastDetectMillis);` |
| 4 | PWM 값이 올바르게 변경되는지 | `startFan()` | `Serial.println(pwmValue);` |
| 5 | 20초 미감지 판정이 올바르게 동작하는지 | `checkNoMotionTimeout()` | `Serial.println("No motion timeout");` |
| 6 | 모터 정지 처리가 실행되는지 | `stopFan()` | `Serial.println("Fan stopped");` |
| 7 | OFF 표시가 올바르게 호출되는지 | `showOff()` | `Serial.println("Display OFF");` |
| 8 | LED 상태가 올바르게 전환되는지 | `updateLed()` | `Serial.println("LED Updated");` |

### Q1: 구현상 문제 확인

> “이 상세설계서에 작성한 함수와 처리 흐름을 바탕으로 Arduino 코드를 작성합니다. 버그가 발생하기 쉬운 부분, 처리 누락, 자료형 문제는 있습니까?”

**AI의 답변(요약):**

기본적으로는 구현 가능한 내용으로 되어 있지만, 몇 가지 버그가 발생하기 쉬운 부분이 있습니다.

pwmValue 는 0~255 범위로 설계되어 있으므로,
Arduino 의 analogWrite() 에 그대로 사용할 수 있습니다.

또한 `millis()` 를 사용하는 시간 계산에서는 `int` 가 아니라 `unsigned long` 을 사용해야 합니다.
특히 20초 판정에서는 `millis() - lastDetectMillis >= stopTime` 형태로 계산하면 안전합니다.

다음으로, `lastDetectMillis` 의 초기값이 0인 상태 그대로라면,
기동 직후에 20초가 경과했다고 판단될 가능성이 있습니다.
따라서 사람을 감지했을 때 확실히 현재 시각으로 업데이트할 필요가 있습니다.

또한 상태 번호가 예상 외의 값이 되었을 경우의 처리도 필요합니다.
상세설계서에서는 이상 시 대기중으로 되돌린다고 작성되어 있으므로,
구현 시에도 `currentState = 0` 으로 되돌리는 처리를 넣는 것이 좋습니다.

마지막으로, `delay()` 를 사용하면 센서 확인이나 표시 업데이트가 멈출 가능성이 있으므로,
PWM 증가와 20초 판정은 `millis()` 를 사용해서 처리할 필요가 있습니다.

위 사항에 주의하면 Arduino 에서 큰 문제 없이 구현할 수 있다고 생각합니다.
