### 3-3. その他の重要ロジック（任意）

【ロジック名】
PIRセンサーの誤検知対策と20秒停止判定

【考え方】
PIRセンサーは人の動きや周囲の環境によって一時的にHIGH/LOWが変化することがある。
そのため、1回LOWになっただけですぐに扇風機を停止せず、最後に人を検知した時刻から20秒経過した場合に停止する。

【処理の流れ】
1. PIRセンサーの値を読み取る
2. 人を検知した場合、lastDetectMillis を現在時刻に更新する
3. 人を検知している間はモーターを動作させ、PWM値を表示する
4. 人を検知しない状態が続いた場合、現在時刻と lastDetectMillis の差を確認する
5. 20秒以上経過した場合、モーターを停止し、7セグメントディスプレイに「OFF」を表示する

【入力値と出力値の関係】
PIRセンサー = HIGH
→ 人を検知
→ モーター動作
→ LED点灯
→ 7セグメントディスプレイにPWM値を表示

PIRセンサー = LOW かつ 20秒未満
→ すぐには停止しない
→ モーター動作を継続
→ PWM値表示を継続

PIRセンサー = LOW かつ 20秒以上
→ 人がいないと判断
→ モーター停止
→ LED消灯
→ 7セグメントディスプレイに「OFF」を表示

```

## 6. AIレビュー記録

> グループレビューの前に必ず実施してください。

### Q1: 実装上の問題確認

> 「この詳細設計書に書いた関数と処理フローをもとに Arduino でコードを書きます。バグになりやすい箇所・処理の抜け・型の問題はありますか？」

**AIの回答（要約）：**


基本的には実装できる内容になっていますが、いくつかバグになりやすい箇所があります。

pwmValue は 0〜255 の範囲で設計されているため、
Arduino の analogWrite() にそのまま使用できる。

また、`millis()` を使う時間計算では、`int` ではなく `unsigned long` を使う必要があります。特に 20秒判定では、`millis() - lastDetectMillis >= stopTime` の形で計算すると安全です。

次に、`lastDetectMillis` の初期値が 0 のままだと、起動直後に20秒経過したと判断される可能性があります。そのため、人を検知したときに確実に現在時刻で更新する必要があります。

さらに、状態番号が想定外の値になった場合の処理も必要です。詳細設計書では異常時に待機中へ戻すと書いてあるため、実装時にも `currentState = 0` に戻す処理を入れるとよいです。

最後に、`delay()` を使うとセンサー確認や表示更新が止まる可能性があるため、PWMの増加や20秒判定は `millis()` を使って処理する必要があります。

以上の点に注意すれば、大きな問題なくArduinoで実装できると考えます。
